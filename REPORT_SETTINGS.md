# Настройки отчетов

## Параметры отчетов в админ-панели

### Период отчетов (REPORT_PERIOD_DAYS)

Определяет, за сколько дней собирать данные для отчета.

**Примеры:**
- `1` - данные за последние 24 часа
- `7` - данные за последнюю неделю
- `30` - данные за последний месяц

**По умолчанию:** 1 день

### Время отправки отчетов (REPORT_TIME)

Время ежедневной отправки отчетов в формате `ЧЧ:ММ`.

**Примеры:**
- `09:00` - отчеты отправляются каждый день в 9:00 утра
- `18:00` - отчеты отправляются каждый день в 6:00 вечера
- `00:00` - отчеты отправляются каждый день в полночь

**Важно:** 
- Если установлено время отправки, отчеты будут отправляться **один раз в день** в указанное время
- Если время не установлено, используется интервал проверки (`SCHEDULER_INTERVAL_MINUTES`)

**По умолчанию:** `09:00`

## Как изменить настройки

1. Откройте админ-панель: http://localhost:8001/admin
2. Перейдите в раздел "Настройки"
3. Измените нужные параметры:
   - **Период отчетов** - укажите количество дней
   - **Время отправки отчетов** - укажите время в формате ЧЧ:ММ
4. Нажмите "Сохранить настройки"

## Исправление расчета сумм

Система теперь использует улучшенный алгоритм расчета сумм заказов:

### Для Ozon:
- Проверяет несколько вариантов структуры данных
- Логирует предупреждения, если сумма не найдена
- Использует детальное логирование для отладки

### Для Wildberries:
- Использует поле `totalPrice` из ответа API
- Обрабатывает ошибки корректно

## Отладка некорректных данных

Если суммы или количество заказов отображаются неверно:

1. **Включите DEBUG логирование** в админке
2. **Проверьте логи** при формировании отчета
3. **Проверьте структуру данных** - логи покажут доступные ключи в заказах

В логах вы увидите:
- Сколько заказов обработано
- Откуда берется сумма для каждого заказа
- Предупреждения, если сумма не найдена

## Примеры использования

### Ежедневные отчеты в 9 утра за последние 24 часа
```
REPORT_PERIOD_DAYS=1
REPORT_TIME=09:00
```

### Еженедельные отчеты в понедельник в 8 утра
```
REPORT_PERIOD_DAYS=7
REPORT_TIME=08:00
```

### Отчеты каждые 2 часа за последние 6 часов
```
REPORT_PERIOD_DAYS=0.25  # 6 часов
SCHEDULER_INTERVAL_MINUTES=120
REPORT_TIME=  # Оставить пустым для использования интервала
```
