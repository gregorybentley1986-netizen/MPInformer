# Админ-панель MPInformer

## Доступ к админ-панели

После запуска приложения админ-панель доступна по адресу:

**http://localhost:8001/admin**

## Аутентификация

По умолчанию используются следующие учетные данные:
- **Имя пользователя:** `admin`
- **Пароль:** `admin123`

⚠️ **Важно:** В продакшене обязательно измените пароль!

### Изменение пароля

Откройте файл `app/admin/auth.py` и измените значение `ADMIN_PASSWORD`:

```python
ADMIN_PASSWORD = "ваш_новый_пароль"
```

Или добавьте переменную окружения в `.env`:
```
ADMIN_PASSWORD=ваш_новый_пароль
```

## Функции админ-панели

### 📊 Главная страница (`/admin`)

- **Статус системы** - отображение текущего состояния планировщика
- **Текущие настройки** - интервал проверки, уровень логирования, порт сервера
- **Быстрые действия:**
  - Перезапустить планировщик
  - Сформировать отчет вручную
  - Перейти к настройкам

### ⚙️ Настройки (`/admin/settings`)

#### Изменение интервала проверки заказов

1. Введите новое значение в минутах (от 1 до 1440)
2. Нажмите "Сохранить настройки"
3. Планировщик автоматически перезапустится с новым интервалом

#### Изменение уровня логирования

Доступные уровни:
- **DEBUG** - Подробная отладочная информация (все события)
- **INFO** - Общая информация (рекомендуется)
- **WARNING** - Только предупреждения и ошибки
- **ERROR** - Только ошибки

### 🔄 Перезапуск планировщика

Если планировщик остановился или работает некорректно:
1. Перейдите на главную страницу админки
2. Нажмите кнопку "🔄 Перезапустить планировщик"

### 📊 Ручное формирование отчета

Для немедленного получения отчета:
1. Перейдите на главную страницу админки
2. Нажмите кнопку "📊 Сформировать отчет вручную"
3. Отчет будет отправлен в Telegram

## Безопасность

⚠️ **Рекомендации по безопасности:**

1. **Измените пароль по умолчанию** перед использованием в продакшене
2. **Используйте HTTPS** при доступе к админке через интернет
3. **Ограничьте доступ** к порту сервера через файрвол
4. **Регулярно проверяйте логи** на предмет подозрительной активности

## Устранение неполадок

### Админка не открывается

1. Проверьте, что приложение запущено
2. Убедитесь, что порт сервера правильный (по умолчанию 8001)
3. Проверьте логи приложения на наличие ошибок

### Изменения не применяются

1. Проверьте, что файл `.env` доступен для записи
2. Убедитесь, что планировщик перезапустился (проверьте логи)
3. Перезапустите приложение вручную, если изменения не применились

### Ошибка аутентификации

1. Проверьте правильность имени пользователя и пароля
2. Убедитесь, что используется правильный формат HTTP Basic Auth
3. Проверьте файл `app/admin/auth.py` на наличие изменений

## API Endpoints

Админ-панель также предоставляет API endpoints:

- `GET /admin` - Главная страница
- `GET /admin/settings` - Страница настроек
- `POST /admin/settings/update` - Обновление настроек
- `GET /admin/scheduler/restart` - Перезапуск планировщика
- `GET /admin/report/manual` - Ручное формирование отчета

Все endpoints требуют HTTP Basic Authentication.
